## Mailbox root decision

- The tracked demo parameter file sets:
  `shared_mailbox_root_template = "{workspace_dir}/shared-mailbox"`
- So the intended mailbox root for an interactive run is inside the chosen workspace, for example:
  `$WORKSPACE_DIR/shared-mailbox`
- Both `start-session` commands should use that exact same mailbox root path.

## Workdir decision

- The demo runner passes the same `--workdir "$WORKSPACE_DIR"` argument to both `start-session` commands.
- So both agents run with the same working directory:
  `$WORKSPACE_DIR`
- This is separate from the mailbox root, which remains:
  `$WORKSPACE_DIR/shared-mailbox`

## Job-dir clarification

- The user asked for the distinction between Houmao `workdir` and per-agent `job_dir` for this demo.
- The demo only passes `--workdir "$WORKSPACE_DIR"` and does not set a local-jobs override.
- Canonical runtime docs and the path resolver show that, in this case, each session gets:
  `<working-directory>/.houmao/jobs/<session-id>/`
- Therefore, for this demo:
  - both agents share the same `workdir`: `$WORKSPACE_DIR`
  - each agent gets its own distinct `job_dir` under:
    `$WORKSPACE_DIR/.houmao/jobs/<session-id>/`
- The exact `<session-id>` values are assigned at launch time and can be read from the `start-session` JSON payload, which also includes the resolved `job_dir`.
