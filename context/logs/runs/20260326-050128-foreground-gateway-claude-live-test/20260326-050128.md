## Problems hit and fixed

### 1. Foreground gateway startup exported broken tmux metadata

Symptom:

- `houmao-mgr agents gateway attach --foreground ...` failed before readiness.
- `gateway.log` showed gateway startup crashing in `GatewayCurrentInstanceV1` validation because `tmux_window_id`, `tmux_window_index`, and `tmux_pane_id` were missing.

Root cause:

- The same-session gateway shell command used literal `'$TMUX_PANE'` inside nested tmux metadata lookups.
- That prevented `$TMUX_PANE` from expanding inside the foreground gateway bootstrap shell.

Fix:

- Updated [runtime.py](/data1/huangzhe/code/houmao/src/houmao/agents/realm_controller/runtime.py) so the foreground bootstrap command uses `"$TMUX_PANE"` when resolving `#{window_id}` and `#{window_index}`.
- Added a regression test in [test_gateway_support.py](/data1/huangzhe/code/houmao/tests/unit/agents/realm_controller/test_gateway_support.py) that checks the command expands the live tmux pane value.

### 2. Foreground attach timed out even though the gateway had already started

Symptom:

- After fixing the broken env export, `attach --foreground` still sometimes failed with:
  `Timed out waiting for gateway startup to publish its bound port.`
- In the live tmux session, the foreground gateway window could already exist and the gateway could already be healthy.

Root cause:

- The attach wait loop treated "tmux pane not discoverable yet" as a blocker and refused to accept a valid `current-instance.json` until the pane had been observed first.
- On the real session, `current-instance.json` could appear before `list_tmux_panes_shared()` exposed the new auxiliary pane.

Fix:

- Updated [runtime.py](/data1/huangzhe/code/houmao/src/houmao/agents/realm_controller/runtime.py) so foreground attach can accept a matching `current-instance.json` and continue health probing before pane discovery catches up.
- Kept the stricter failure behavior for panes that disappear after they have already been observed.
- Added regression tests in [test_gateway_support.py](/data1/huangzhe/code/houmao/tests/unit/agents/realm_controller/test_gateway_support.py) covering:
  - initial pane discovery delay
  - valid current-instance publication before pane visibility
