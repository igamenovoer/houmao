## Result

The implementation removed the sustained product false-ready blocks found in Kimi ST03 and Codex ST05. Canonical current-code replay reduced exact UC-03 label differences from 1,225 of 11,202 samples (10.93%) to 190 of 11,202 (1.70%), an 84.5% reduction. Of the remaining 190 exact-label differences, 104 change the downstream ready-versus-blocked decision. None of those prompt-admission differences forms a sustained interval of at least one second.

This is a detector regression result, not full behavioral UC-03 qualification. The retained reference was generated by `operator/manual_label_rubric.py`; its label artifact has no independent human author, reviewer, or rubric digest. The revised comparator therefore names it `legacy_derived_state_labels` and accepts separately supplied reviewer-approved direct labels when available.

## Source Check and Corrected Diagnosis

The original observation was correct: important readiness classifications were wrong. Its proposed reasons were only partly correct.

- Kimi Code queue surfaces were a real detector omission. Local source at `extern/orphan/kimi-code` commit `f17a6ecb` shows that `QueuePaneComponent` renders the three retained-message hints only while a message is streaming or compacting. `kimi-tui.ts` queues a submitted message whenever the streaming phase is not idle. A current queue pane is therefore busy evidence even when the editor remains empty.
- Kimi moon and braille spinners were already correctly treated as activity by the detector. The legacy rubric instead labeled short moon-spinner intervals as ready or draft. Those residual comparisons are reference defects, not evidence that the tracker should become ready.
- Codex pending-input surfaces were a real detector omission. Local source at `extern/orphan/codex` commit `5c19155c` documents `PendingInputPreview` as the display for pending steers, rejected steers, and queued messages while a turn is in progress. The 0.144.1 recording uses newer section headings that the detector did not recognize.
- Codex ST04 samples `s000097..s000136` visibly show the `Select Model and Effort` selector. The old report's overlay-versus-draft diagnosis was reversed: the tracker had failed to identify the selector, while the legacy rubric classified the `› 1.` selection row as a text draft.
- Codex ST05 restart samples showed two distinct conditions. Runtime loss around `s001028` was valid `tui_down` evidence. The real restart bug was the later live-process window that retained the old Codex prompt above a new shell launch command before new provider chrome rendered.

## Implemented Fixes

- Kimi 0.23.x now extracts all three bounded queue hints and makes a current queue block ready posture and success.
- Codex 0.144.x now recognizes every maintained pending-input header, treats those headers as non-response cells, and keeps the turn busy when the normal status row is displaced.
- Codex reconnect matching now accepts only bounded reconnect status shapes, so arbitrary prompt prose containing `retry` does not create activity.
- Codex current list selectors, including the model-and-effort selector, now produce explicit blocking-overlay evidence.
- Codex and Kimi ignore retained prompt rows above a newer shell launch command until fresh provider chrome appears below that boundary.
- The long-horizon raw gate now recognizes Kimi moon/braille activity, every Kimi queue mode, and Codex pending-input sections. A submitted operation must also show a post-submit progress edge before two stable ready polls can complete it.
- The UC-03 comparator replays the detector code in the current checkout from raw recordings. It applies diagnostics before surface rows, requires affirmative overlay evidence, separates direct labels from legacy generated references, and reports exact-label and prompt-admission mismatch intervals separately.

## Residual Inspection

Canonical replay has one sustained exact-label interval: Codex ST04 `s000097..s000136` for 3.919 seconds. The recording visibly shows the model selector. Current tracking reports `busy_overlay`; the legacy reference reports `busy_draft`. Both classifications block prompt admission, and the source-backed overlay classification is the correct one.

At 10 Hz, Codex ST05 has one additional 1.0-second exact-label interval around the prompt beginning with `/model`. Current tracking conservatively reports unknown editor classification while the legacy reference reports draft. Both remain blocked, so this is not an admission inversion. It falls below one second in the canonical timestamps and disappears as a sustained interval at 5 Hz and 2 Hz.

The remaining prompt-admission differences are short intervals at sampled transitions or known legacy-reference defects:

- Kimi ST03 and ST04 residual false-busy comparisons coincide with visible moon activity that the source-backed tracker correctly blocks but the legacy rubric labels ready or draft.
- Codex ST01, ST03, and ST05 differences are isolated prompt/status transitions; the longest canonical interval is under one second.
- Claude ST02's three exact differences are active-versus-draft boundary labels and do not change prompt admission.

No remaining inspected interval supports widening a detector rule or restoring a ready claim. Doing so would reintroduce the false-ready behavior that blocks safe gateway prompt admission.
