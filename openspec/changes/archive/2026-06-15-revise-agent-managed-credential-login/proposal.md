## Why

Agents that run credential login commands directly can lose the interactive browser, device-code, or paste-back surface needed to complete provider authentication. Claude credential guidance also does not reflect Claude Code's current long-lived `CLAUDE_CODE_OAUTH_TOKEN` setup-token lane, so agents may offer a less suitable default when the user simply asks to create a new Claude credential.

## What Changes

- Teach `houmao-credential-mgr` login guidance to start provider login helper commands in a dedicated tmux session so agents and users can attach, inspect, and interact with the login process.
- Require login guidance to preserve ambient proxy variables in the tmux login session by default, including uppercase and lowercase `HTTP_PROXY`, `HTTPS_PROXY`, `ALL_PROXY`, and `NO_PROXY` variants when present.
- Clarify that Houmao's supported credential `login` command still owns temporary provider home creation, provider CLI invocation, artifact import, and cleanup; tmux is only the operator interaction wrapper around the maintained command.
- Revise Claude credential-kind guidance so the preferred unspecified Claude creation path is a long-lived Claude Code token generated by `claude setup-token` and stored through `--oauth-token` / `CLAUDE_CODE_OAUTH_TOKEN`.
- Place Claude vendor-login/OAuth import as the fallback when the user cannot or does not want to use the long-lived token lane, while keeping `ANTHROPIC_AUTH_TOKEN` scoped to explicit bearer-token or gateway use.

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `houmao-manage-credentials-skill`: change the packaged credential-management skill requirements for agent-managed login execution, proxy inheritance, and Claude credential-kind defaulting.

## Impact

- Affected packaged skill assets under `src/houmao/agents/assets/system_skills/houmao-credential-mgr/`, especially `actions/login.md`, `actions/add.md`, and `references/claude-credential-kinds.md`.
- No expected changes to the `houmao-mgr` credential CLI contract; the change uses the existing `credentials <tool> login`, `--oauth-token`, and related credential flags.
- Test impact is documentation/asset validation for the packaged system skill text and affected OpenSpec requirements.
