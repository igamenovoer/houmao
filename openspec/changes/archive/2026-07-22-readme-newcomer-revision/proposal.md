## Why

The repository README is accurate and thorough, but it is written for readers who already understand Houmao. A first-time reader hits eight or more undefined concepts (`managed agent`, `gateway sidecar`, `mailbox identity`, `turn evidence`, `system-skill installation target`) in the second paragraph, meets installation variants before knowing what a skill pack is, and reaches a roughly 40-line System Skills section that reads as internal architecture notes rather than onboarding. At the same time, the README must not fix readability by gutting its identity: the operator has directed that the name, metaphor, design rationale, and system architecture remain first-class sections with richer narrative, not casualties of brevity. The existing `readme-structure` spec mandates a concise System Skills summary the current README violates, lacks newcomer-readability requirements, and carries stale details (Kimi Code 0.11.0 guidance, `houmao-touring` references, a ~20-line intro cap incompatible with the expanded narrative).

## What Changes

- Expand the opening narrative **in place, before Quick Start**: the name-origin story stands alone as brand identity (never a teaching device and never a source of synonyms like "strands" or "clones" for `specialist`/`managed agent`), and "Why This Design" grows from bullets into short rationale paragraphs (real CLI processes, no central orchestrator, full provider capability). The pre-Quick-Start prose budget rises from ~20 to ~60 lines with subheadings.
- Rewrite "Architecture at a Glance" into a walkthrough: narrate the existing team diagram and add a second mermaid diagram showing one managed agent's anatomy (provider CLI process in tmux, gateway sidecar, mailbox identity, memory directory).
- Define every load-bearing concept (specialist, project profile, managed agent, gateway, mailbox, loop, `$houmao-*` invocation) in plain language at or before first use, independent of the brand narrative.
- Restructure the Quick Start into one golden path (prerequisites, install, `tmux` check, `npx skills add …`, start CLI agent, `$houmao-admin-welcome start-guided-tour`) where each step states its expected outcome and the first `You:`/`AI:` exchange appears inline. The welcome skill is taught as the single skill invocation every newcomer must remember. Alternative install paths become one-line doc pointers.
- Collapse the System Skills section to the mandated concise summary (six-skill table, short actor-model paragraph, welcome named as the starting skill) and move pack-membership, route-selection, ownership, and compatibility prose to `docs/getting-started/system-skills-overview.md`.
- Split long sentences (several exceed 40–100 words) and replace abstraction-nouns newcomers cannot guess (`posture`, `surface` as a noun, `credential lane`, `pack closure`) with concrete wording.
- Give the embedded demo video a caption; add a license note.
- Update the `readme-structure` spec: add newcomer-readability and narrative requirements, replace the ~20-line intro cap with the expanded-narrative budget, and refresh stale requirement text (current Kimi Code guidance, welcome-based tour entry).

No code behavior changes. No breaking changes.
