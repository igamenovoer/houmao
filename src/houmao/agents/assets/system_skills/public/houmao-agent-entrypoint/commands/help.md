# Help

## Workflow

1. **Keep help read-only** and skip identity verification, target discovery, sibling loading, and command execution.
2. **Explain the actor posture** and the exact fresh self-identity command required before substantive work.
3. **Summarize eligible self and peer routes** and identify `$houmao-shared-routines`, `$houmao-agent-loop-pro`, and `$houmao-agent-loop-lite` as top-level sibling destinations.
4. **Show invocation forms** and state that there is no managed-agent welcome route.

If the help request does not map cleanly to these steps, use the native planning tool to build a bounded read-only response from the entrypoint identity, route, target, and sibling contracts, then answer without execution.
