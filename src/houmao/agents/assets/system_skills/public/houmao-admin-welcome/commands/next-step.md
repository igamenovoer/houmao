# Next Step

## Workflow

1. **Read the current guided-path stage and posture** without changing either.
2. **Identify one smallest useful continuation** that preserves existing project and runtime state.
3. **Explain why it fits** and name any unresolved required decision.
4. **Return a read-only recommendation or context-preserving admin-entrypoint handoff**.

If the request does not map cleanly to these steps, use the native planning tool to build a bounded read-only continuation from current state, guided paths, and the user's goal, then recommend one next step.

Use the current guided-path stage and known posture to offer one small read-only next step. If the user wants execution, build the handoff from [../references/admin-handoff.md](../references/admin-handoff.md).
