# Choose Path

## Workflow

1. **Read supplied intent and posture** without mutating project or runtime state.
2. **Compare the curated paths** and select the narrowest fit for the desired outcome and current state.
3. **Explain the fit** and distinguish required inputs from optional refinements.
4. **Offer one next orientation step or exact admin-entrypoint handoff** with context preserved.

If the request does not map cleanly to these steps, use the native planning tool to build a bounded read-only path comparison from the curated paths, supplied context, and user goal, then recommend one path.

Match the user's stated outcome and visible project posture to one curated path. State why the path fits, identify required and optional inputs, and offer either another read-only teaching step or an exact admin-entrypoint handoff.
