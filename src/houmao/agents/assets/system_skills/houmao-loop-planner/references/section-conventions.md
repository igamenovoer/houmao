# Section Conventions

Use this reference to keep the Markdown bundle structured without exploding it into many small policy files.

## `participants.md`

Use one section per participant with these headings:

- `Role`
- `Receives From`
- `Reports To`
- `May Call`
- `Work Artifacts`
- `Must Send`
- `Escalate When`
- `Must Not`

## `execution.md`

Use these headings:

- `Loop Kind`
- `Topology Summary`
- `Message Flow`
- `Master Procedure`
- `Reporting Posture`
- `Completion Behavior`
- `Stop Behavior`

## `distribution.md`

Use these headings:

- `Send To Participants`
- `Expected Confirmations`
- `Before Start`
- `Next Runtime Skill`
